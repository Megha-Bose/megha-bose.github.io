Automata Theory & FSMs - Formal Models of What Machines Can Compute

Aug 3, 2021
8 min read

When you type a URL into your browser’s address bar, a tiny program runs to check whether the string you typed is even syntactically valid. That program is a finite state machine. Every regular expression you’ve ever written - \d+, [a-z]*, (foo|bar)+ - compiles to a finite automaton before it runs. The traffic light controller at the intersection outside your building is a finite state machine. The vending machine that accepts quarters is a finite state machine. The protocol handshake that begins every HTTPS connection is a finite state machine.

Automata theory is the theory of what computation means. It starts with the simplest model imaginable - a machine with no memory - and asks: what can such a machine compute? The answer turns out to be surprisingly rich, and the limits turn out to be just as revealing as the capabilities.

The Simplest Possible Machine

Imagine a machine that reads a string of symbols one at a time, left to right. At each moment, the machine is in one of a fixed set of states. When it reads a symbol, it transitions to a new state based only on the current state and the symbol - nothing else. At the end of the string, the machine either accepts or rejects.

That’s it. No memory, no stack, no ability to go back and re-read. Just: current state, read symbol, move to next state, repeat.

This is a deterministic finite automaton (DFA).

Let’s build one. Suppose we want to recognize all binary strings that end in 01. We need three states:

  • $q_0$: nothing useful has happened yet (or the last symbol was 1 after something other than 0)
  • $q_1$: the last symbol was 0
  • $q_2$: the last two symbols were 01 - this is our accept state

The transitions:

Current state Read 0 Read 1
$q_0$ $q_1$ $q_0$
$q_1$ $q_1$ $q_2$
$q_2$ $q_1$ $q_0$

Try the string 1001. We start at $q_0$: read 1 → $q_0$, read 0 → $q_1$, read 0 → $q_1$, read 1 → $q_2$. We end in the accept state. The string ends in 01. It works.

Try 100. Read 1 → $q_0$, read 0 → $q_1$, read 0 → $q_1$. End in $q_1$: not an accept state. Correct - 100 does not end in 01.

Formally, a DFA is a 5-tuple $M = (Q, \Sigma, \delta, q_0, F)$ where $Q$ is the finite set of states, $\Sigma$ is the alphabet, $\delta: Q \times \Sigma \to Q$ is the transition function, $q_0 \in Q$ is the start state, and $F \subseteq Q$ is the set of accept states. The machine runs on input $w = w_1 w_2 \cdots w_n$ by computing the sequence of states $q_0, \delta(q_0, w_1), \delta(\delta(q_0, w_1), w_2), \ldots$, accepting if the final state is in $F$.

The model is maximally simple - one state variable, one character of input at a time, no lookahead, no memory of anything beyond the current state.

Nondeterminism: Multiple Worlds

Designing DFAs can be cumbersome. Sometimes you want to say: “at this point, the machine might be in one of several possible situations.” NFAs let you say exactly that.

A nondeterministic finite automaton (NFA) allows two things DFAs don’t:

  1. From any state on any symbol, you can move to multiple states simultaneously - the machine “splits” and runs all branches in parallel.
  2. You can take $\varepsilon$-transitions - move to a new state without reading any input.

The NFA accepts a string if any branch of the computation reaches an accept state.

NFAs are much easier to design. Want an NFA that accepts strings containing 01 as a substring? Make a three-state machine: start state $q_0$ loops on everything, then on reading 0 nondeterministically also sends a thread to $q_1$, then on reading 1 from $q_1$ move to accept state $q_2$ which loops there. The nondeterminism handles the “try starting the match here” logic automatically.

You might think: NFAs sound more powerful than DFAs, since they can be in multiple states at once. But they’re not. The same class of languages. Here’s why.

The Subset Construction

Every NFA can be converted to an equivalent DFA. The idea is elegant: if the NFA can be in any subset of its states at once, make each possible subset into a single DFA state.

Given an NFA with states ${q_0, q_1, q_2}$, the DFA has up to $2^3 = 8$ states: $\emptyset$, ${q_0}$, ${q_1}$, ${q_2}$, ${q_0, q_1}$, ${q_0, q_2}$, ${q_1, q_2}$, ${q_0, q_1, q_2}$.

The DFA starts at the state corresponding to the $\varepsilon$-closure of the NFA’s start state (all states reachable from $q_0$ without reading any input). On reading symbol $a$ from DFA state $S$ (which represents NFA being in all states in $S$), the DFA moves to the state corresponding to: all NFA states reachable from any state in $S$ on symbol $a$, plus the $\varepsilon$-closure of those.

This construction always works. The resulting DFA recognizes exactly the same language as the NFA. Therefore NFA and DFA are equivalent in power - they recognize exactly the same class of languages, called the regular languages.

Discomfort check. The subset construction turns an NFA with $n$ states into a DFA with up to $2^n$ states. Does it always blow up this badly? In the worst case, yes - and there exist specific NFA families where every equivalent DFA genuinely needs exponentially many states. But in practice, most NFA constructions (especially those arising from compiling regular expressions) produce DFAs far smaller than the worst case. The exponential is a theoretical worst case, not a practical one.

Regular Expressions

You’ve used regular expressions. The theory says: every regex corresponds to an NFA, and every NFA corresponds to a regex.

Formally, a regular expression over alphabet $\Sigma$ is built from:

  • Single symbols: a, b, …
  • The empty string: $\varepsilon$
  • Union: $R_1 \mid R_2$ - strings matching either
  • Concatenation: $R_1 R_2$ - a string matching $R_1$ followed by one matching $R_2$
  • Kleene star: $R^*$ - zero or more strings each matching $R$, concatenated

Every regular expression can be converted to an NFA (by structural induction on the expression), and every NFA can be converted back to a regular expression. So all three models - DFA, NFA, regular expression - define exactly the same class of languages.

This is Kleene’s theorem, and it’s the first major equivalence result in the theory of computation.

What Regular Languages Cannot Do

The machines we’ve built so far have no memory. They can’t count. And that limitation shows up clearly when you try to recognize a language that requires counting.

Consider the language ${0^n 1^n : n \geq 0}$ - strings of $n$ zeros followed by $n$ ones: $\varepsilon$, 01, 0011, 000111, and so on. Can a DFA recognize this?

No. Here’s why intuitively: after reading a hundred zeros, a DFA is in some state. But that state encodes all the information the machine has. If the DFA has $k$ states, after reading $k+1$ zeros, two of those inputs must have put the machine in the same state (by the pigeonhole principle). But then the machine treats inputs of different lengths of zeros the same - and it can’t correctly match them against the corresponding number of ones.

This argument is formalized as the Pumping Lemma.

Pumping Lemma. If $L$ is regular, then there exists a “pumping length” $p \geq 1$ such that any string $w \in L$ with $|w| \geq p$ can be split into three parts $w = xyz$ where:

  1. $|y| \geq 1$ (the “pumped” part is nonempty)
  2. $|xy| \leq p$ (the pump is near the beginning)
  3. $xy^i z \in L$ for all $i \geq 0$ (you can repeat the middle any number of times and stay in $L$)

The proof is just the pigeonhole argument: if $p = |Q|$ (the number of DFA states), any string of length $\geq p$ forces the machine to visit some state twice, creating a loop that can be repeated.

To prove ${0^n 1^n}$ is not regular: suppose it were, with pumping length $p$. Take $w = 0^p 1^p$. The pumping lemma says we can write $w = xyz$ with $|xy| \leq p$ and $|y| \geq 1$. Since $|xy| \leq p$, the substring $y$ lies entirely within the block of zeros: $y = 0^k$ for some $k \geq 1$. But then $xy^2z = 0^{p+k} 1^p$, which has more zeros than ones - not in the language. Contradiction.

FSMs can’t count. This single insight motivates everything that comes next.

What You Can Build With FSMs

Despite their limitations, finite automata appear everywhere computation happens fast.

Lexers. A compiler’s first pass tokenizes its input into keywords, identifiers, numbers, and operators. This is done by a single DFA constructed from the union of regular expressions for all token types - fast, linear-time scanning.

Pattern matching. The Knuth-Morris-Pratt string search algorithm is essentially a DFA over the pattern string, achieving $O(n)$ search time with no backtracking.

Network packet filtering. Firewall rules are DFAs over byte streams, running at line speed.

Protocol state machines. The TCP handshake, TLS negotiation, and every other network protocol is specified as a state machine and often verified formally as one.

Summary

Model Key feature Expressiveness Conversion
DFA One state at a time, deterministic Regular languages Canonical form
NFA Multiple states in parallel, $\varepsilon$-transitions Regular languages $\leq 2^n$ DFA states via subset construction
Regex Compositional text notation Regular languages NFA by structural induction
Pumping Lemma Loop in long strings Test for NON-regularity N/A

All three machine models (DFA, NFA, regex) are equivalent: they recognize exactly the regular languages. The regular languages are closed under union, concatenation, Kleene star, complement, and intersection. Languages like ${0^n 1^n}$ that require counting exceed the power of finite automata - for those, we need a more powerful model.
